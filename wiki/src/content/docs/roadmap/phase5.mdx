## Phase 5 Architecture

### Declarative System Configuration

The centerpiece of Phase 5 is the `system.yaml` manifest file that defines the entire system state:

```yaml
# /etc/systemd/system/system.yaml
system:
  version: "0.4.12"
  target: "declarative-base"
  
  user:
    username: "admin"
    password: "$6$..."
    shell: "/bin/bash"
    groups: ["wheel", "audio", "video"]
    homed_storage: "subvolume"
    
  network:
    hostname: "ouroboros"
    enable_networkd: true
    enable_iwd: true
    enable_resolved: true
    
  disk:
    device: "/dev/vda"
    use_luks: false
    btrfs_label: "ouroborOS"
    swap_type: "zram"
    
  locale:
    locale: "en_US.UTF-8"
    keymap: "us"
    timezone: "America/Santiago"
    
  desktop:
    profile: "minimal"
    dm: "auto"
    
  security:
    secure_boot: false
    sbctl_include_ms_keys: false
    
  extra_packages: []
  post_install_action: "reboot"
```

### Package Management

#### Atomic Package Updates

Enhanced `our-pac` with atomic operations:

```bash
# Atomic package update workflow
our-pac update
# Steps:
# 1. Create snapshot: our-snapshot create pre-update
# 2. Remount read-write: btrfs property set / ro false
# 3. Update packages: pacman -Syu
# 4. Re-sign boot files: sbctl sign-all
# 5. Sync boot entries: bootctl update
# 6. Prune snapshots: our-snapshot prune
```

```python
# Proposed desired state engine
class DesiredStateEngine:
    def __init__(self):
        self.current_state = self.load_current_state()
        self.desired_state = self.load_desired_state()
        
    def reconcile(self):
        differences = self.compare_states()
        return self.apply_changes(differences)
        
    def verify(self):
        return self.current_state == self.desired_state
```
