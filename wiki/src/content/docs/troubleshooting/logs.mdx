# Logs & Debugging

ouroborOS uses systemd's journal as the primary logging system. Here's how to find what you need.

## Essential Commands

```bash
# View all logs (newest first)
journalctl -e

# Follow logs in real-time
journalctl -f

# Filter by service
journalctl -u ouroboros-health.service
journalctl -u our-pac.service

# Filter by boot (previous boot)
journalctl -b -1

# Filter by priority (errors and worse)
journalctl -p err
```

## System Health Check

```bash
# Run the built-in health check
ouroboros-health --doctor

# Machine-readable output
ouroboros-health --json
```

:::note
`ouroboros-health` checks: root RO status, failed units, machine-id, system.yaml validity, Btrfs pool usage, snapshot count, orphan boot entries, SMART status, Secure Boot, TPM2, and available updates.
:::

## Installer Logs

### Installation Process Logs

The ouroborOS installer creates detailed logs throughout the installation process:

```bash
# Main installer log (includes all states and operations)
cat /tmp/ouroborOS-install.log

# Filter by installation phase
grep "NETWORK_SETUP" /tmp/ouroborOS-install.log
grep "CONFIGURE" /tmp/ouroborOS-install.log

# View error logs only
grep -i error /tmp/ouroborOS-install.log

# Check installation checkpoints
cat /tmp/ouroborOS-checkpoints/last_checkpoint.json
```

### Debug Installation Issues

If the installer fails, check the logs for troubleshooting:

```bash
# View the last 100 lines of installer output
tail -n 100 /tmp/ouroborOS-install.log

# Filter for specific operations that failed
grep "pacstrap\|configure\|bootloader" /tmp/ouroborOS-install.log

# Check for disk operation errors
grep "disk.sh\|format\|mount" /tmp/ouroborOS-install.log
```

## Snapshot Metadata

### Inspect Snapshot Details

Each snapshot stores metadata for debugging and recovery:

```bash
# List all snapshots with metadata
sudo our-snapshot list

# View detailed snapshot information
sudo our-snapshot info 2026-04-18T143000

# View raw Btrfs properties
btrfs property get /.snapshots/2026-04-18T143000 ro
btrfs subvolume show /.snapshots/2026-04-18T143000

# Check snapshot creation time
stat /.snapshots/2026-04-18T143000
```

### Boot Entry Analysis

```bash
# List all systemd-boot entries
bootctl list

# View specific boot entry
cat /boot/loader/entries/ouroborOS.conf
cat /boot/loader/entries/ouroboros-snapshot-2026-04-18T143000.conf

# Check boot entry validity
bootctl status

# Verify root subvolume paths
findmnt -n -o OPTIONS / | grep subvol
```

## Advanced Debugging Techniques

### systemd Journal Analysis

```bash
# View all logs with human-readable timestamps
journalctl -xe

# Filter by specific time range
journalctl --since "2026-04-18 14:30:00" --until "2026-04-18 15:00:00"

# Show logs from specific boot
journalctl -b -1 -u systemd-networkd

# Filter by process
journalctl _PID=1

# Show kernel messages
journalctl -k

# Export logs for debugging
journalctl > system-logs.txt
journalctl -u sshd > sshd-logs.txt
```

### Btrfs System Debugging

```bash
# Check filesystem integrity
sudo btrfs check --readonly /

# Show filesystem usage and statistics
btrfs filesystem df /
btrfs filesystem usage /

# List all subvolumes
btrfs subvolume list /

# Show subvolume information
btrfs subvolume show /

# Balance filesystem status
btrfs balance status /

# Scrub for bad blocks
sudo btrfs scrub start /
```

## Emergency Logging

### If System Fails to Boot

```bash
# 1. Boot from live ISO
# 2. Mount root partition
mount -t btrfs -o subvol=@ /dev/sdX2 /mnt

# 3. Access logs from mounted system
cat /mnt/tmp/ouroborOS-install.log
cat /mnt/var/log/journal/*/system.journal | journalctl -o short-precise

# 4. Check snapshot metadata
ls -la /mnt/.snapshots/
cat /mnt/.snapshots/.metadata/
```

### Save Debug Information

```bash
# Create debug bundle
mkdir debug-$(date +%Y%m%d-%H%M%S)
cd debug-$(date +%Y%m%d-%H%M%S)

# Copy system information
cp /tmp/ouroborOS-install.log .
sudo journalctl > system.journal
sudo btrfs filesystem df / > filesystem-stats.txt
sudo btrfs subvolume list / > subvolume-list.txt
bootctl list > boot-entries.txt
sudo ouroboros-health --json > health.json

# Archive debug bundle
tar -czf debug-$(date +%Y%m%d-%H%M%S).tar.gz *
```

## Log Rotation and Management

### Configure Log Retention

```bash
# View current journal settings
journalctl --verify --vacuum-size=100M
journalctl --disk-usage

# Set retention policy
sudo journalctl --vacuum-size=500M
sudo journalctl --vacuum-time=2weeks
```

### Backup Important Logs

```bash
# Backup critical logs
mkdir -p /var/backups/logs
cp /tmp/ouroborOS-install.log /var/backups/logs/
cp /var/log/pacman.log /var/backups/logs/

# Rotate logs monthly
logrotate --state /var/log/logrotate.state /etc/logrotate.conf
```

## Kernel Log Analysis

### dmesg Kernel Messages

```bash
# View kernel ring buffer
dmesg
dmesg -T  # Show human-readable timestamps

# Follow kernel messages in real-time
dmesg -w

# Filter by specific subsystems
dmesg -T | grep -i "network"
dmesg -T | grep -i "usb"
dmesg -T | grep -i "pci"

# Filter by severity
dmesg -l err    # Errors only
dmesg -l warn   # Warnings and errors
dmesg -l info   # Info and above

# Search for specific patterns
dmesg | grep "eth0"
dmesg | grep "driver"
```

### Kernel Module Analysis

```bash
# List loaded modules
lsmod
lsmod | grep -i network

# Show module information
modinfo e1000e
modinfo iwlmvm

# Load/unload modules (debugging)
sudo modprobe -v e1000e
sudo rmmod e1000e

# Check module dependencies
modinfo --depmod e1000e
```

### Network Driver Debugging

### Network Stack Debugging

### Advanced Kernel Debugging


## Recovery Procedures

### Corrupted Btrfs Filesystem

```bash
# Check filesystem integrity (mount read-only first)
mount -o ro /dev/sda2 /mnt
btrfs scrub start /mnt

# If scrub finds errors, try repair
btrfs scrub start -B /mnt

# Check device stats for hardware errors
btrfs device stats /mnt
```

### Failed Boot Recovery

```bash
# Boot from ISO, mount the system
mount -o subvol=@ /dev/sda2 /mnt

# Check bootloader entries
ls /mnt/loader/entries/

# Verify boot configuration
cat /mnt/loader/loader.conf

# Reinstall bootloader if needed
arch-chroot /mnt bootctl install
```
