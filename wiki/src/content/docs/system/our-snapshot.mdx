# our-snapshot — Btrfs Snapshots

`our-snapshot` is ouroborOS's powerful Btrfs snapshot management tool that provides comprehensive control over system snapshots, enabling atomic rollbacks, automated pruning, and system integrity verification. It's the backbone of ouroborOS's immutable architecture, ensuring every system state is preserved and reversible.

## What is our-snapshot?

`our-snapshot` transforms Btrfs snapshot management from a complex manual process into an intuitive, automated system. It handles the complexity of Btrfs subvolumes, snapshot naming, and lifecycle management while providing powerful tools for system recovery and maintenance.

```bash
# Traditional Btrfs operations - complex
btrfs subvolume snapshot / /.snapshots/manual-$(date +%Y%m%d)
btrfs subvolume list /mnt
btrfs subvolume delete /.snapshots/old-snapshot

# our-snapshot - simple and powerful
our-snapshot create manual-backup
our-snapshot list
our-snapshot prune --keep 30
```

## Core Architecture

### **Snapshot Categories**
our-snapshot organizes snapshots into logical categories:

- **Automatic**: Created during package operations (`pacop-YYYY-MM-DDTHHMMSS/`)
- **Manual**: Created by users (`YYYY-MM-DD_LABEL/`)
- **System**: Core system snapshots (`install/`, `upgrade/`, etc.)
- **Golden**: The immutable baseline snapshot (never pruned)

```yaml
# Snapshot hierarchy
/.snapshots/
├── install/                    # Golden snapshot (never deleted)
├── 2024-01-15T10:00:00_upgrade/
├── 2024-01-15T10:30:00_docker_install/
├── 2024-01-15T14:00:00_manual_backup/
└── .metadata/
    ├── snapshot_list.json
    ├── snapshot_info.json
    └── system_state.json
```

### **Metadata Management**
```json
// /.snapshots/.metadata/snapshot_info.json
{
  "snapshots": [
    {
      "id": "2024-01-15T10:00:00",
      "label": "upgrade",
      "parent": "install",
      "size": "2.5GB",
      "created": "2024-01-15T10:00:00Z",
      "description": "System package upgrade",
      "type": "automatic"
    }
  ]
}
```

## Basic Operations

### **Creating Snapshots**
```bash
# Create manual snapshot
our-snapshot create my-custom-config

# Create snapshot with description
our-snapshot create "Pre-update backup" --description "Before major system update"

# Create snapshot in specific category
our-snapshot create --category backup my-backup

# Create recursive snapshot of specific subvolume
our-snapshot create --subvolume @home home-backup
```

### **Listing Snapshots**
```bash
# List all snapshots
our-snapshot list

# List recent snapshots
our-snapshot list --limit 10

# List snapshots by category
our-snapshot list --category automatic

# List snapshots with detailed information
our-snapshot list --verbose

# List snapshots in JSON format
our-snapshot list --json
```

### **Deleting Snapshots**
```bash
# Delete specific snapshot
our-snapshot delete 2024-01-15T10:00:00

# Delete snapshot by label
our-snapshot delete --label upgrade

# Delete multiple snapshots
our-snapshot delete 2024-01-15T10:00:00 2024-01-15T11:00:00

# Delete all snapshots except golden and recent
our-snapshot delete --all --keep-install --keep 10
```

### **Pruning Snapshots**
```bash
# Keep only last 30 snapshots
our-snapshot prune --keep 30

# Keep snapshots by age
our-snapshot prune --keep-days 7

# Keep golden snapshot + last 10
our-snapshot prune --keep-install --keep 10

# Dry run to see what would be pruned
our-snapshot prune --dry-run --keep 30
```

### **System Integration**

### **Boot Management**

## Configuration Options

### **Configuration File**
```yaml
# /etc/ouroborOS/our-snapshot.conf
snapshot:
  prefix: "snapshot"
  keep_minimum: 10
  keep_automatic: 30
  keep_manual: 50
  keep_golden: true
  compression: "zlib"
  encryption: false

pruning:
  strategy: "age"
  keep_days: 30
  keep_count: 50
  schedule: "daily"
  
logging:
  level: "info"
  file: "/var/log/our-snapshot.log"
  rotate: "weekly"
```

### **Global Configuration**
```bash
# Set global snapshot prefix
our-snapshot config set snapshot.prefix "backup"

# Set retention policy
our-snapshot config set pruning.keep_days "7"

# Enable compression
our-snapshot config set snapshot.compression "zstd"
```

## Snapshot Types and Usage

### **Automatic Snapshots**
```bash
# Created during package operations
our-pac upgrade → creates: /.snapshots/pacop-YYYY-MM-DDTHHMMSS/
our-pac install nginx → creates: /.snapshots/pacop-YYYY-MM-DDTHHMMSS_nginx_install/

# Automatic snapshot naming
# Format: {operation}_{optional_detail}_{timestamp}
# Examples:
# - pacop-2024-01-15T10:30:00_system_upgrade
# - pacop-2024-01-15T10:45:00_docker_install
# - pacop-2024-01-15T11:00:00_neovim_update
```

### **Manual Snapshots**
```bash
# User-created snapshots
our-snapshot create "My custom config"
# Creates: /.snapshots/2024-01-15T11:15:00_My custom config/

# Category-based snapshots
our-snapshot create --category config desktop-setup
# Creates: /.snapshots/2024-01-15T11:30:00_desktop-setup/
```

### **System Snapshots**
```bash
# System-critical snapshots (never pruned)
install/         # Initial system installation
upgrade/        # Major system upgrades
recovery/       # System recovery snapshots
golden/         # Baseline snapshot
```

## Integration with System

### **Package Manager Integration**
```bash
# Automatic snapshots during package operations
our-pac upgrade
# Automatically creates: /.snapshots/pacop-YYYY-MM-DDTHHMMSS/

# Manual snapshot before package operation
our-snapshot create pre-custom-package

```

### **System Recovery**
```bash

our-rollback now

```

### **Monitoring and Alerting**
```bash

```

## Troubleshooting

### **Common Issues**
```bash

# Rebuild snapshot metadata
```

### **Error Recovery**
```bash
# Restore from corrupted snapshot
our-snapshot restore 2024-01-15T10:00:00 --force

# Recover deleted snapshot
our-snapshot recover-deleted

# Emergency snapshot creation
our-snapshot create --emergency emergency-restore
```

```bash

# Balance snapshot distribution
our-snapshot balance --profile=ssd

# Compress old snapshots
our-snapshot compress --age 30d
```
