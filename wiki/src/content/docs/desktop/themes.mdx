# Desktop Themes and Customization

orooborOS provides comprehensive theming capabilities across all supported desktop environments. Whether you prefer a minimal aesthetic or a highly customized workspace, the system supports easy theme management and consistent styling.

## Overview

The theming system in ouroborOS is designed to be:

- **Consistent**: All desktop environments follow similar color palette conventions
- **Customizable**: Easy to modify colors, icons, and layouts
- **Persistent**: Theme changes are preserved across reboots and updates
- **Cross-desktop**: Works across Hyprland, Niri, GNOME, and KDE

:::note

Theme management is integrated with the `our-pac` package manager. When you update system packages, your theme customizations are preserved automatically.

:::

## Color Palette

### Default Color Scheme

orooborOS uses an emerald/teal color palette that provides a fresh, modern appearance:

```yaml
# Default color scheme
colors:
  primary: "#10b981"    # Emerald (dark mode accent)
  secondary: "#059669" # Dark Emerald (light mode accent)  
  background: "#0f172a" # Very Dark (header background)
  surface: "#1e293b"    # Card backgrounds
  text: "#f1f5f9"       # Primary text
  text-secondary: "#94a3b8" # Secondary text
```

### Theme Customization

You can customize colors by modifying theme files in your home directory:

```bash
# Theme configuration directory
~/.config/ouroborOS/themes/

# Create custom theme
mkdir -p ~/.config/ouroborOS/themes/my-theme
```

## Desktop-Specific Theming

### Hyprland Themes

Hyprland configuration is located at `~/.config/hypr/hyprland.conf`:

```ini
# Hyprland theme configuration
$accent = #10b981
$background = #0f172a
$surface = #1e293b

windowrule = float, ^(firefox)$
windowrule = tile, ^(kitty)$

general {
    border_size = 2
    col.active_border = $accent
    col.inactive_border = #334155
}
```

### Niri Themes

Niri configuration is in `~/.config/niri/config.kdl`:

```kdl
// Niri theme configuration
$accent: #10b981;
$background: #0f172a;
$surface: #1e293b;

windowRules {
    match app_id "firefox" {
        floating true
    }
}
```

### GNOME Themes

GNOME themes are managed through extensions:

```bash
# Install GNOME tweaks

# Theme installation
gnome-extensions install user-themes@gnome-shell-extensions.gnome.org
```

### KDE Themes

KDE customization through System Settings:

```bash

```

## Dynamic Theme Switching

orooborOS supports automatic theme switching based on time of day:

```yaml
# ~/.config/ouroborOS/theme-switcher.yaml
theme:
  day:
    colors:
      primary: "#059669"  # Light emerald
      background: "#ffffff"
  night:
    colors:
      primary: "#10b981"  # Dark emerald  
      background: "#0f172a"
```

## Theme Backup and Migration

### Backup Current Theme Configuration

```bash
# Create theme backup
theme-backup() {
    local backup_dir="~/.themes-backup/$(date +%Y%m%d-%H%M%S)"
    mkdir -p "$backup_dir"
    
    # Backup desktop-specific configs
    cp -r ~/.config/hypr/ "$backup_dir/hypr" 2>/dev/null || true
    cp -r ~/.config/niri/ "$backup_dir/niri" 2>/dev/null || true
    cp -r ~/.config/gtk-*.0/ "$backup_dir/gtk" 2>/dev/null || true
    cp -r ~/.config/plasma* "$backup_dir/plasma" 2>/dev/null || true
    cp -r ~/.local/share/gnome* "$backup_dir/gnome-shell" 2>/dev/null || true
    
    echo "Theme backup saved to: $backup_dir"
}

# Backup current theme
theme-backup
```

### Migrate Theme Between Systems

```bash
# Export theme configuration
theme-export() {
    local output_dir="theme-$(date +%Y%m%d)"
    mkdir -p "$output_dir"
    
    # Collect theme files
    cp ~/.config/hypr/hyprland.conf "$output_dir/" 2>/dev/null || true
    cp ~/.config/niri/config.kdl "$output_dir/" 2>/dev/null || true
    cp ~/.config/gtk-3.0/settings.ini "$output_dir/" 2>/dev/null || true
    cp ~/.config/gtk-4.0/settings.ini "$output_dir/" 2>/dev/null || true
    cp -r ~/.themes/* "$output_dir/themes/" 2>/dev/null || true
    cp -r ~/.icons/* "$output_dir/icons/" 2>/dev/null || true
    
    # Create archive
    tar -czf "$output_dir.tar.gz" "$output_dir"
    rm -rf "$output_dir"
    
    echo "Theme exported to: $output_dir.tar.gz"
}

# Export theme
theme-export
```

:::caution

Always backup your theme configurations before major system updates to prevent customization loss.

:::
