# systemd-networkd Configuration

systemd-networkd is the default network management system in ouroborOS, providing robust, reliable network configuration using declarative configuration files. It replaces traditional NetworkManager with a more systemd-integrated approach.

## Overview

systemd-networkd provides:

- **Declarative network configuration** through .network and .netdev files
- **Automatic network detection** and connection management
- **Integration with systemd** for service dependencies
- **DNS and NTP management** through systemd-resolved and systemd-timesyncd
- **Bonding and teaming** for network redundancy

:::note

ouroborOS uses systemd-networkd as the primary network manager, with iwd for WiFi and systemd-resolved for DNS resolution. This provides a consistent, systemd-native networking experience.

:::

## Installation and Setup

The network stack is pre-installed with ouroborOS. Verify setup:

```bash
# Check network service status
systemctl status systemd-networkd
systemctl status systemd-resolved
systemctl status systemd-timesyncd

# Show network interfaces
ip link show
```

## Configuration Directory

Network configuration files are stored in `/etc/systemd/network/`:

```bash
# List network configurations
ls -la /etc/systemd/network/

# Show network directory structure
tree /etc/systemd/network/
```

## Default Network Configuration

### ouroborOS Default Files

ouroborOS comes with pre-configured network files. Here are the actual configurations:

#### Wired Network (`/etc/systemd/network/20-wired.network`)

```ini
[Match]
Name=en*

[Network]
DHCP=yes
DNS=1.1.1.1
DNS=9.9.9.9

[DHCP]
RouteMetric=10
UseDNS=true
```

#### Wireless Network (`/etc/systemd/network/25-wireless.network`)

```ini
[Match]
Name=wl*

[Network]
DHCP=yes
DNS=1.1.1.1
DNS=9.9.9.9
IgnoreCarrierLoss=3s

[DHCP]
RouteMetric=20
UseDNS=true
```

#### DNS Resolution (`/etc/systemd/resolved.conf`)

```ini
[Resolve]
DNS=1.1.1.1 9.9.9.9
FallbackDNS=8.8.8.8
DNSOverTLS=opportunistic
DNSSEC=allow-downgrade
```

### Custom Configuration

You can add custom network configurations by creating additional `.network` files:

#### Static IP Configuration

```ini
# /etc/systemd/network/10-static.network
[Match]
Name=enp0s3

[Network]
Address=192.168.1.100/24
Gateway=192.168.1.1
DNS=8.8.8.8 8.8.4.4
```

#### Multiple DNS Servers

```ini
# /etc/systemd/network/30-dns.network
[Match]
Name=enp0s3

[Network]
DNS=8.8.8.8
DNS=8.8.4.4
DNS=1.1.1.1
DNS=9.9.9.9
```

## Advanced Configuration Options

### Bonding (Link Aggregation)

```ini
# /etc/systemd/network/40-bond.netdev
[NetDev]
Name=bond0
Kind=bond
MACAddress=00:11:22:33:44:55

[Bond]
Mode=802.3ad
LACPRate=fast
MIIMonitorSec=100
UpDelaySec=2000
DowndelaySec=2000
```

```ini
# /etc/systemd/network/50-bond.network
[Match]
Name=bond0

[Network]
Address=192.168.1.100/24
Gateway=192.168.1.1
DNS=8.8.8.8
```

### VLAN Configuration

```ini
# /etc/systemd/network/60-vlan.netdev
[NetDev]
Name=vlan10
Kind=vlan
Interface=eth0
Id=10

[VLAN]
VLANId=10
```

### Bridge Configuration

```ini
# /etc/systemd/network/80-bridge.netdev
[NetDev]
Name=br0
Kind=bridge
MACAddress=00:11:22:33:44:55

[Bridge]
STP=no
ForwardDelaySec=0
```

## Network Management Commands

### Service Management

```bash
# Enable and start network services
sudo systemctl enable systemd-networkd
sudo systemctl start systemd-networkd

# Enable and start DNS resolution
sudo systemctl enable systemd-resolved
sudo systemctl start systemd-resolved

# Enable and start time sync
sudo systemctl enable systemd-timesyncd
sudo systemctl start systemd-timesyncd

# Enable WiFi (if iwd is configured)
sudo systemctl enable iwd.service
```

### Network Status

```bash
# Show network connections
networkctl

# Show detailed network status
networkctl status

# Show network addresses
networkctl address

# Show DNS configuration
systemd-resolve --status

# Show wireless status (if iwd is enabled)
iwctl station wlan0 show
```

### Network Configuration Reload

```bash
# Reload network configurations
sudo systemctl reload systemd-networkd

# Restart network service
sudo systemctl restart systemd-networkd

# Restart DNS resolution
sudo systemctl restart systemd-resolved

# Show network logs
journalctl -u systemd-networkd -f

# Show DNS logs
journalctl -u systemd-resolved -f
```

## DNS Configuration

### systemd-resolved Setup

```bash
# Create symlink for DNS
ln -sf /run/systemd/resolve/stub-resolv.conf /etc/resolv.conf

systemd-resolve --cache-status
```

### Custom NTP Configuration

```ini
# /etc/systemd/timesyncd.conf
[Time]

FallbackNTP=time.google.com time.windows.com
PollIntervalMinSec=32
PollIntervalMaxSec=2048
```

```bash
# Add basic rules
sudo nft add table inet filter
```


## Notes

- **Network priority**: Wired connections have metric 10, wireless have metric 20
- **DNS resolution**: Uses systemd-resolved with DoT (DNS over TLS)
- **WiFi management**: Handled by iwd, integrated with networkd
- **Time sync**: Uses systemd-timesyncd with NTP pool
- **Service dependencies**: Network services are properly ordered and dependent
