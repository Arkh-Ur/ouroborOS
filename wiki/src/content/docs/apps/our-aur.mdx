# AUR Helper (our-aur)

`our-aur` is a containerized AUR (Arch User Repository) package manager that safely handles AUR packages using systemd-nspawn containers and systemd-sysext. This approach provides isolation, security, and seamless integration with the ouroborOS package management system.

## Overview

The `our-aur` tool provides:

- **Containerized AUR management** using systemd-nspawn
- **Seamless pacman integration** with consistent interface
- **Automatic snapshot protection** before AUR operations
- **Secure package signing** with sbctl
- **Clean rollback capabilities** for failed installations

:::tip

All AUR operations run in isolated containers, preventing system contamination and providing a clean rollback mechanism if packages fail to build or install.

:::

## Installation

The AUR helper is pre-installed with ouroborOS. Verify installation:

```bash
our-aur --version
our-aur --help
```

## Basic Usage

### Package Search

```bash
# Search for packages in AUR
our-aur search neovim

# Search with description
our-aur search --desc "video editor"

# Search maintained packages only
our-aur search --maintained telegram-desktop
```

### Package Installation

```bash
# Install a single package
our-aur install neovim

# Install multiple packages
our-aur install neovim lazygit zellij

# Install with dependencies
our-aur install --deps visual-studio-code-bin
```

### Package Removal

```bash
# Remove a package (keep dependencies)
our-aur remove neovim

# Remove package and dependencies
our-aur remove --nosave neovim

# Remove package and all dependencies
our-aur remove --recursive neovim
```

## AUR Build Process

### Container Architecture

`our-aur` uses a sophisticated container architecture for safe AUR package building:

```bash
# Build process flow
our-aur -S neovim

# 1. Creates ephemeral container
ls /var/tmp/our-aur/containers/
# Output: build-12345 (temporary nspawn container)

# 2. Downloads paru-bin
ls /var/lib/our-aur/paru-bin.pkg.tar.zst
# paru-bin auto-downloaded from GitHub releases

# 3. Builds as non-root user (aurbuild UID 1000)
systemd-nspawn -D /var/tmp/our-aur/containers/build-12345 \
  --user=aurbuild --setenv=HOME=/home/aurbuild

# 4. Installs as systemd-sysext extension
ls /var/lib/extensions/our-aur-neovim/
# Extension appears at /usr/ via overlay
```

### Build Environment

```bash
# Container configuration
cat /var/tmp/our-aur/containers/build-12345/etc/sudoers.d/aurbuild
# Output: aurbuild ALL=(ALL) NOPASSWD: /usr/bin/pacman

# User setup in container
cat /var/tmp/our-aur/containers/build-12345/etc/passwd
# Output: aurbuild:x:1000:1000::/home/aurbuild:/bin/bash

# Network isolation
# Container inherits host networking but runs as isolated user
```

### Package Tracking System

```bash
# Tracking database location
ls /var/lib/our-aur/packages/
# Output: neovim.json lazygit.json zellij.json

# View package tracking
cat /var/lib/our-aur/packages/neovim.json
# Output:
{
  "name": "neovim",
  "version": "0.9.4",
  "aur_url": "https://aur.archlinux.org/packages/neovim",
  "installed_at": "2024-01-15T14:30:00",
  "pkg_names": ["neovim", "neovim-packer"],
  "sysext_path": "/var/lib/extensions/our-aur-neovim"
}

# List tracked packages with metadata
our-aur -Q
# Output: neovim 0.9.4 2024-01-15T14:30:00
#         lazygit 0.15.0 2024-01-14T09:15:00
```

## Security Features

### Container Isolation

```bash
# Container runs as non-root user (aurbuild UID 1000)
# No root privileges inside container

# Container is ephemeral - destroyed after build
# No persistent state between builds

# Builds run in private systemd-nspawn namespace
# No access to host /dev, /proc, /sys except via explicit mounts
```

### Package Verification

```bash
# paru automatically verifies package signatures
# PGP keys handled by pacman keyring

# Build process is transparent
# Watch all build output in real-time:
our-aur -S neovim  # Shows entire build process
```

### Backup and Migration

```bash
# Export tracked packages
mkdir -p ~/backup/our-aur
cp -r /var/lib/our-aur/packages ~/backup/our-aur/

# Export installed extensions
for ext in /var/lib/extensions/our-aur-*; do
    [[ -d "$ext" ]] && cp -r "$ext" ~/backup/our-aur/extensions/
done

# On fresh system, restore tracking
cp -r ~/backup/our-aur/packages /var/lib/our-aur/

# Extensions work automatically via systemd-sysext
```

## Troubleshooting Guide

### Common Build Failures

```bash
# Fix dependency issues
our-aur -S --sync-deps neovim  # Install dependencies first

# Check build environment
our-aur -S --check-env neovim  # Verify system requirements

# View container logs
journalctl -u systemd-nspawn -u aurbuild

# Debug build step-by-step
# 1. Check container creation
ls /var/tmp/our-aur/containers/
# 2. Check paru download
ls /var/lib/our-aur/paru-bin.pkg.tar.zst
# 3. Check build logs in container
systemd-nspawn -D /var/tmp/our-aur/containers/build-12345 -- bash
```

### Network Issues

```bash
# Test AUR connectivity
curl -I https://aur.archlinux.org/rpc/v5/search/neovim

# Test package download
curl -I https://github.com/Morganamilo/paru/releases/latest/download/paru-x86_64.pkg.tar.zst

# Reset network stack
systemctl restart systemd-networkd iwd systemd-resolved
```

### Extension Issues

```bash
# Check systemd-sysext status
systemctl status systemd-sysext

# Force extension refresh
systemd-sysext merge

# Check extension loading errors
journalctl -u systemd-sysext

# Remove corrupted extension
our-aur -R neovim
rm -rf /var/lib/extensions/our-aur-neovim
our-aur -S neovim  # Reinstall
```

### System Compatibility

```bash
# Check architecture compatibility
uname -m  # Should be x86_64

# Check base system compatibility
pacman -Q base-devel

# Verify systemd-sysext availability
systemctl status systemd-sysext
```
