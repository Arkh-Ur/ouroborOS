# FIDO2/WebAuthn Management (our-fido2)

`our-fido2` is a comprehensive FIDO2/WebAuthn authentication tool supporting multiple authenticator types including USB, BLE GATT, and Hybrid QR (CTAP2). It provides secure passwordless authentication for ouroborOS systems and web applications.

## Overview

The FIDO2 manager provides:

- **Multi-protocol support** for USB, BLE, and QR authentication
- **Cross-platform compatibility** with FIDO2 standards
- **Secure credential management** with backup and recovery
- **Web application integration** for seamless authentication
- **System integration** with login managers and services

:::note

our-fido2 requires BlueZ with `--experimental` flag enabled for BLE functionality and the Advertising Monitor API. USB and QR authentication work with standard FIDO2 devices.

:::

## Installation

The FIDO2 manager is pre-installed with ouroborOS. Verify installation:

```bash
our-fido2 --version
our-fido2 --help
```

## Basic Usage

### Device Management

```bash
# List connected FIDO2 devices
our-fido2 list

# Show device details
our-fido2 info "My YubiKey"

# Check device capabilities
our-fido2 capabilities "My YubiKey"

# Test device connection
our-fido2 test "My YubiKey"
```

### Credential Management

```bash
# Create new credential
our-fido2 credential create "My Account" --device "My YubiKey"

# List credentials
our-fido2 credential list

# Remove credential
our-fido2 credential remove "My Account"

# Backup credentials
our-fido2 credential backup > credentials-backup.json
```

## PAM Integration

our-fido2 integrates FIDO2 tokens into the Pluggable Authentication Modules (PAM) stack for system authentication, enabling passwordless sudo, TTY login, and SSH authentication.

### System-Wide Authentication

```bash
# Register FIDO2 token for system authentication (sudo/login)
sudo our-fido2 pam register --system

# Enable FIDO2 for sudo authentication
sudo our-fido2 pam enable sudo

# Enable FIDO2 for all services (sudo, login, SSH)
sudo our-fido2 pam enable all

# Check PAM configuration status
sudo our-fido2 pam status
```

### User-Specific Authentication

```bash
# Register FIDO2 token for current user (session unlock)
our-fido2 pam register

# Enable FIDO2 for SSH authentication only
sudo our-fido2 pam enable ssh

# Remove FIDO2 authentication from specific service
sudo our-fido2 pam disable sudo
```

### PAM Workflow

```bash
# 1. Register token (touch when it blinks)
sudo our-fido2 pam register --system

# 2. Enable authentication service
sudo our-fido2 pam enable sudo

# 3. Test sudo authentication
sudo -k  # Clear cached credentials
sudo whoami  # Touch token when prompted
```

## SSH FIDO2 Keys

Generate and manage FIDO2 SSH keys where the private key material stays securely on the hardware token.

### Key Generation

```bash
# Generate standard FIDO2 SSH key
our-fido2 ssh generate

# Generate resident key (stored on token)
our-fido2 ssh generate --resident

# Generate key with custom path and application
our-fido2 ssh generate --file ~/.ssh/custom_key --application "ssh:my-server"

# List existing FIDO2 SSH keys
our-fido2 ssh list
```

### Resident Key Management

```bash
# Load resident keys from token
our-fido2 ssh load-resident

# Use resident keys across different machines
# (Keys are stored on token, can be loaded anywhere)
our-fido2 ssh load-resident

# Test SSH with FIDO2 key
ssh -i ~/.ssh/id_ed25519_sk user@remote-host
```

## LUKS2 Disk Encryption

Unlock encrypted LUKS2 partitions using FIDO2 tokens instead of or in addition to passphrases.

### Enroll FIDO2 for Disk Unlock

```bash
# Enroll FIDO2 token in LUKS2 partition
sudo our-fido2 luks enroll /dev/sda2

# List enrolled key slots
sudo our-fido2 luks list /dev/sda2

# Remove FIDO2 enrollment (keeps passphrase slot)
sudo our-fido2 luks unenroll /dev/sda2
```

### LUKS2 Authentication Workflow

```bash
# 1. Enroll token during installation
sudo our-fido2 luks enroll /dev/vda2

# 2. At boot, touch the token when prompted
# Token must be present during each boot

# 3. Fallback passphrase still works
# Can be used if token is lost or unavailable
```

```bash

# 2. System Authentication (sudo)
```

:::danger

⚠️ **Security Warning**: Always backup your FIDO2 credentials regularly. Losing access to your FIDO2 device without backup credentials may permanently lock you out of your accounts.

- For resident keys, write down recovery codes when available
- For PAM authentication, keep backup authentication methods configured
- For LUKS2, always keep the passphrase as a fallback
- Store device backups in secure, offline locations

:::
