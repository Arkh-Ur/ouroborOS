# Temas y Personalización del Escritorio

ouroborOS proporciona capacidades completas de theming en todos los entornos de escritorio soportados. Ya sea que prefieras una estética minimalista o un espacio de trabajo altamente personalizado, el sistema admite una fácil gestión de temas y estilos consistentes.

## Resumen

El sistema de theming en ouroborOS está diseñado para ser:

- **Consistente**: Todos los entornos de escritorio siguen convenciones similares de paleta de colores
- **Personalizable**: Fácil de modificar colores, iconos y diseños
- **Persistente**: Los cambios de tema se preservan a través de reinicios y actualizaciones
- **Multi-entorno**: Funciona en Hyprland, Niri, GNOME y KDE

:::note

La gestión de temas está integrada con el gestor de paquetes `our-pac`. Cuando actualizas paquetes del sistema, tus personalizaciones de tema se preservan automáticamente.

:::

## Paleta de Colores

### Esquema de Colores Predeterminado

ouroborOS utiliza una paleta de colores esmeralda/teal que proporciona una apariencia fresca y moderna:

```yaml
# Esquema de colores predeterminado
colors:
  primary: "#10b981"    # Esmeralda (acento modo oscuro)
  secondary: "#059669" # Esmeralda Oscuro (acento modo claro)  
  background: "#0f172a" # Muy Oscuro (fondo de encabezado)
  surface: "#1e293b"    # Fondo de tarjetas
  text: "#f1f5f9"       # Texto primario
  text-secondary: "#94a3b8" # Texto secundario
```

### Personalización de Temas

Puedes personalizar colores modificando archivos de tema en tu directorio home:

```bash
# Directorio de configuración de temas
~/.config/ouroborOS/themes/

# Crear tema personalizado
mkdir -p ~/.config/ouroborOS/themes/my-theme
```

## Themado Específico por Entorno de Escritorio

### Hyprland

La configuración de Hyprland se encuentra en `~/.config/hypr/hyprland.conf`:

```ini
# Configuración de tema Hyprland
$accent = #10b981
$background = #0f172a
$surface = #1e293b

windowrule = float, ^(firefox)$
windowrule = tile, ^(kitty)$

general {
    border_size = 2
    col.active_border = $accent
    col.inactive_border = #334155
}
```

### Niri

La configuración de Niri está en `~/.config/niri/config.kdl`:

```kdl
// Configuración de tema Niri
$accent: #10b981;
$background: #0f172a;
$surface: #1e293b;

windowRules {
    match app_id "firefox" {
        floating true
    }
}
```

### GNOME

Los temas de GNOME se gestionan a través de extensiones:

```bash
# Instalar GNOME tweaks
our-pac install gnome-tweaks

# Instalación de temas
gnome-extensions install user-themes@gnome-shell-extensions.gnome.org
```

### KDE

Personalización de KDE a través de System Settings:

```bash
# Gestión de temas KDE

```

## Cambio Dinámico de Temas

ouroborOS admite cambio automático de tema basado en la hora del día:

```yaml
# ~/.config/ouroborOS/theme-switcher.yaml
theme:
  day:
    colors:
      primary: "#059669"  # Esmeralda claro
      background: "#ffffff"
  night:
    colors:
      primary: "#10b981"  # Esmeralda oscuro  
      background: "#0f172a"
```

## Respaldo y Migración de Temas

### Respaldo de Configuración de Tema Actual

```bash
# Crear respaldo de tema
theme-backup() {
    local backup_dir="~/.themes-backup/$(date +%Y%m%d-%H%M%S)"
    mkdir -p "$backup_dir"
    
    # Respaldar configs específicos de escritorio
    cp -r ~/.config/hypr/ "$backup_dir/hypr" 2>/dev/null || true
    cp -r ~/.config/niri/ "$backup_dir/niri" 2>/dev/null || true
    cp -r ~/.config/gtk-*.0/ "$backup_dir/gtk" 2>/dev/null || true
    cp -r ~/.config/plasma* "$backup_dir/plasma" 2>/dev/null || true
    cp -r ~/.local/share/gnome* "$backup_dir/gnome-shell" 2>/dev/null || true
    
    echo "Respaldo de tema guardado en: $backup_dir"
}

# Respaldar tema actual
theme-backup
```

### Migrar Tema Entre Sistemas

```bash
# Exportar configuración de tema
theme-export() {
    local output_dir="theme-$(date +%Y%m%d)"
    mkdir -p "$output_dir"
    
    # Recopilar archivos de tema
    cp ~/.config/hypr/hyprland.conf "$output_dir/" 2>/dev/null || true
    cp ~/.config/niri/config.kdl "$output_dir/" 2>/dev/null || true
    cp ~/.config/gtk-3.0/settings.ini "$output_dir/" 2>/dev/null || true
    cp ~/.config/gtk-4.0/settings.ini "$output_dir/" 2>/dev/null || true
    cp -r ~/.themes/* "$output_dir/themes/" 2>/dev/null || true
    cp -r ~/.icons/* "$output_dir/icons/" 2>/dev/null || true
    
    # Crear archivo
    tar -czf "$output_dir.tar.gz" "$output_dir"
    rm -rf "$output_dir"
    
    echo "Tema exportado a: $output_dir.tar.gz"
}

# Exportar tema
theme-export
```
